# SimpleWebApp

A Phoenix application for testing the [`html_to_pdf`](https://github.com/tauspace/html_to_pdf) library. It renders an employee report as an HTML page — including a base64-encoded PNG image — and lets you download the same content as a PDF by clicking a button.

The PDF is generated by sending the HTML to a running [Gotenberg](https://gotenberg.dev) instance via `html_to_pdf`. This makes it a good end-to-end test for verifying that Gotenberg is reachable, that base64-embedded images survive the conversion, and that the library's multipart request building works as expected.

## Environment variables

| Variable | Required | Default | Description |
|---|---|---|---|
| `GOTENBERG_URL` | Yes | — | Base URL of your Gotenberg instance, e.g. `http://localhost:3000`. The app appends `/forms/chromium/convert/html` to this URL when generating a PDF. |
| `GOTENBERG_TIMEOUT` | No | `30000` | Request timeout in milliseconds for the Gotenberg HTTP call. Increase this if you are converting large or complex pages. |

Set them before starting the server:

```bash
export GOTENBERG_URL=http://localhost:3000
export GOTENBERG_TIMEOUT=60000
mix phx.server
```

Or inline for a one-off run:

```bash
GOTENBERG_URL=http://localhost:3000 mix phx.server
```

## Running Gotenberg

If you do not have a Gotenberg instance, the quickest way to start one locally is with Docker:

```bash
docker run --rm -p 3000:3000 gotenberg/gotenberg:8
```

## Setup and start

```bash
mix setup       # install deps and build assets
mix phx.server  # start on http://localhost:4000
```

Or run inside IEx:

```bash
iex -S mix phx.server
```

Visit [localhost:4000](http://localhost:4000), then click **Download PDF** to trigger a conversion through `html_to_pdf`.
